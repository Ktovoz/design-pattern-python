# 中介者模式 (Mediator Pattern)

## 简介
中介者模式是一种行为型设计模式，它用一个中介对象来封装一系列的对象交互。中介者使各对象不需要显式地相互引用，从而使其耦合松散，而且可以独立地改变它们之间的交互。

## 适用场景
- 一组对象以定义良好但是复杂的方式进行通信
- 一个对象引用其他很多对象并且直接与这些对象通信，导致难以复用该对象
- 想定制一个分布在多个类中的行为，而又不想生成太多的子类

## 优点
- 降低类的复杂度
- 降低了对象之间的耦合度
- 将各同事类解耦

## 缺点
- 中介者对象可能会变得过于复杂
- 系统的灵活性降低
