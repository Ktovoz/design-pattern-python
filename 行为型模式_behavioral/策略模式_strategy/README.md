# 策略模式 (Strategy Pattern)

## 简介
策略模式是一种行为型设计模式，它定义了一系列算法，并将每个算法封装起来，使它们可以互换。

## 适用场景
- 当需要在运行时改变算法或行为时
- 当使用条件语句选择不同的行为时
- 当需要在不同情况下使用不同的算法时

## 优点
- 算法可以自由切换
- 避免使用多重条件语句
- 提供了开放-封闭原则的完美例子

## 缺点
- 客户端必须了解不同策略的区别
- 策略类的数量会增加
