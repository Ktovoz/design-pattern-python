# 观察者模式 (Observer Pattern)

## 简介
观察者模式是一种行为型设计模式，它定义了对象之间的一对多依赖关系，当一个对象的状态发生改变时，所有依赖于它的对象都会得到通知并被自动更新。

## 适用场景
- 当一个对象的改变需要同时改变其他对象时
- 当不知道有多少个对象需要改变时
- 当一个对象必须在其他对象更新时更新自己，但又不知道其他对象是谁时

## 优点
- 支持广播通信
- 符合开闭原则
- 降低耦合度

## 缺点
- 如果观察者很多，通知会花费很多时间
- 如果被观察者和观察者之间有循环依赖，可能会导致系统崩溃
