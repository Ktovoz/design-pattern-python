# 访问者模式 (Visitor Pattern)

## 简介
访问者模式是一种行为型设计模式，它允许你将算法从它所操作的数据结构中分离出来。

## 适用场景
- 一个对象结构包含很多类对象，它们有不同的接口，而想对这些对象实施一些依赖于其具体类的操作
- 需要对一个对象结构中的对象进行很多不同的并且不相关的操作，而需要避免污染这些类的接口
- 需要对一个对象结构中的每个元素执行一些操作，但要避免改变这些类的接口

## 优点
- 符合单一职责原则
- 优秀的扩展性
- 分离了算法和对象

## 缺点
- 具体元素对访问者公布细节
- 违反了依赖倒置原则
- 增加新的元素类很困难
