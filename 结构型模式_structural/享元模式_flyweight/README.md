# 享元模式 (Flyweight Pattern)

## 简介
享元模式是一种结构型设计模式，它使用共享对象来有效支持大量细粒度的对象。

## 适用场景
- 当一个应用程序使用了大量的对象时
- 当完全由于使用大量的对象，造成很大的存储开销时
- 当对象的大多数状态都可变为外部状态时

## 优点
- 大大减少对象的创建
- 减轻内存压力
- 提高运行效率

## 缺点
- 增加了系统的复杂性
- 违背了单一职责原则
